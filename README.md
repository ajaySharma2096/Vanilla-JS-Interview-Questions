# Frontend Interview Questions in Vanilla JavaScript

A curated collection of frontend interview coding problems solved in **Vanilla JavaScript**. This repository is focused on common interview-style questions that help strengthen core JavaScript logic, DOM handling, problem-solving, and frontend fundamentals without relying on external libraries or frameworks.

## Overview

This repository contains solutions to multiple problems commonly asked in **frontend developer interviews**, implemented using plain JavaScript. The goal is to practice and demonstrate a strong understanding of:

- Core JavaScript concepts
- Problem-solving approaches
- DOM manipulation
- Event handling
- Array and object operations
- String manipulation
- Functional programming basics
- Writing clean and understandable code

These solutions are useful for:
- frontend interview preparation
- JavaScript revision
- practice with pure JavaScript
- understanding logic building without frameworks

## Why Vanilla JavaScript?

Vanilla JavaScript helps build a strong foundation in frontend development. Many interviewers ask candidates to solve problems without frameworks to evaluate:

- logical thinking
- JavaScript fundamentals
- code clarity
- ability to work with browser-native APIs
- understanding of DOM behavior and events

This repository is intended to help practice those essential skills.

## How to Use

### 1. Clone the repository

```bash
git clone https://github.com/ajaySharma2096/Vanilla-JS-Interview-Questions.git
```

### 2. Open the project

```bash
cd Vanila-JS
```

### 3. Run the code

You can run JavaScript files in different ways depending on the problem:

#### Using the browser
If a solution includes DOM-related code, open the corresponding HTML file in a browser.

#### Using Node.js
For pure JavaScript logic problems:

```bash
node filename.js
```
